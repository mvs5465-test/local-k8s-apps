# Local K8s Applications

ArgoCD `Application` definitions grouped by sync wave. Pair with [`local-k8s-argocd`](https://github.com/mvs5465-test/local-k8s-argocd), which reads these manifests via an `ApplicationSet` and generates the live applications.

Apps in `apps/wave-0/`, `apps/wave-1/`, and `apps/wave-2/` are enabled. Apps in `apps/disabled/` are kept in git but are not generated by the ArgoCD `ApplicationSet`.

## Repo Layout

- `apps/wave-0/`: cluster entrypoint and prerequisites
- `apps/wave-1/`: platform services and shared observability backends
- `apps/wave-2/`: user-facing apps and higher-level integrations
- `apps/disabled/`: manifests intentionally kept out of the generated live set
- `values/`: Helm value overrides used by the app manifests
- `manifests/`: supporting Kubernetes resources that are not ArgoCD `Application` objects

## Storage Configuration

Data is stored under `~/clusterstorage/` with the following structure:
```
~/clusterstorage/
тФФтФАтФА files/              тЖР Jellyfin media files (nginx file server)
```

See `local-k8s-argocd` README for Colima mount configuration.

## Adding Apps

1. Create `<app-name>-app.yaml` in the appropriate `apps/wave-*/` directory
2. Reference the chart or git source you want ArgoCD to sync
3. If the app pulls its chart from a git repo, add that repo URL to `local-k8s-argocd/manifests/config/appproject.yaml`
4. Merge to `main` and let the bootstrap `ApplicationSet` regenerate the live `Application`

## Toggling Apps

- Enable an app by placing its manifest in `apps/wave-0/`, `apps/wave-1/`, or `apps/wave-2/`
- Disable an app by moving its manifest to `apps/disabled/`
- ArgoCD only generates apps from the wave directories, so moving a manifest to `apps/disabled/` removes it from the live set on the next sync

## Supporting Manifests

This repo also keeps a small set of non-Application resources that support the cluster-level app setup:

- `manifests/external-secrets/`: secret stores and secret wiring
- `manifests/servicemonitors/`: Prometheus scrape targets
- `manifests/cluster-health-reporter/`: optional Slack heartbeat job
- `manifests/image-restart-controller/`: optional image refresh restart job

## Development

Create feature branches to test changes. ArgoCD watches `main`, so changes sync automatically after merge.

See `CLAUDE.md` for development guidelines.
