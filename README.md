# Local K8s Applications

ArgoCD `Application` definitions split between a small bootstrap layer and the normal auto-syncing cluster app set. Pair with [`local-k8s-argocd`](https://github.com/mvs5465-test/local-k8s-argocd), which reads these manifests via `ApplicationSet` resources and generates the live applications.

Apps in `apps/bootstrap-wave-0/` and `apps/bootstrap-wave-1/` are the prerequisite layer for first boot. Apps in `apps/cluster-wave-1/` and `apps/cluster-wave-2/` are the steady-state auto-syncing app set. Apps in `apps/disabled/` are kept in git but are not generated by ArgoCD.

## Storage Configuration

Data is stored under `~/clusterstorage/` with the following structure:
```
~/clusterstorage/
└── files/              ← Jellyfin media files (nginx file server)
```

See `local-k8s-argocd` README for Colima mount configuration.

## Adding Apps

1. Create `<app-name>-app.yaml` in the appropriate enabled app directory
2. Reference your Helm chart or git repo
3. Push to main and let the `ApplicationSet` resources regenerate the live `Application`

## Toggling Apps

- Enable an app by placing its manifest in one of the bootstrap or cluster wave directories
- Disable an app by moving its manifest to `apps/disabled/`
- ArgoCD only generates apps from the enabled wave directories, so moving a manifest to `apps/disabled/` removes it from the live set on the next sync

## Development

Create feature branches to test changes. ArgoCD watches main, so changes sync automatically after merge.

See `CLAUDE.md` for development guidelines.
