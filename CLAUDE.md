# Local K8s Applications - Claude Guidelines

This repo contains ArgoCD Application definitions. Pair with `local-k8s-argocd` infrastructure repo.

## Key Points

- **Watched by**: ArgoCD `ApplicationSet` resources from `local-k8s-argocd`
- **Update frequency**: As needed for new/modified applications
- **Structure**: bootstrap prerequisite waves plus steady-state cluster waves under `apps/`; keep day-2 config like `external-secrets-config` in the cluster waves
- **Commit style**: Same Conventional Commits as main repo

## Common Tasks

### Add new application
1. Create `<name>-app.yaml` in the appropriate bootstrap or cluster wave directory under `apps/`
2. Reference your application git repo
3. If the app sources its Helm chart from a **git repository** (not a public Helm repo), you must whitelist that repo URL in `local-k8s-argocd/manifests/config/appproject.yaml` under `sourceRepos`
4. Push to main → ArgoCD auto-discovers

**Example:** loki-mcp-server pulls its Helm chart from `https://github.com/mvs5465-test/loki-mcp-server`, so that URL was added to sourceRepos.

### Test application changes
1. Create feature branch
2. Modify Application manifests
3. Test locally or in dev environment
4. Merge to main when ready

## Integration with local-k8s-argocd

The bootstrap and steady-state `ApplicationSet` resources in `local-k8s-argocd/manifests/applicationsets/cluster-apps.yaml` point here.

Do NOT manually apply applications - they're auto-generated by the `ApplicationSet` resources.
